Objects:

FILE

Represents a File object

Field Data Type Notes
id Long The ID of the file.
sha256 String A non-null 64 bit hex String with the file's sha256 hash.
sha3_384 String A nullable String with the file's sha3_384 hash.
classification String A nullable String with the malware class (NULL IF POTENTIALLY NOT MALICIOUS).
mime_type String A nullable String with the file MIME type (if known).
addition_date Long A non-null long with the Epoch Millisecond of when this entry was added to the database.
detection_date Long A non-null long with the Epoch Millisecond of when this entry was either first detected in the wild (if known) or the addition date.
sample_available Boolean A non-null Boolean if there is a copy of the file locally.

ROLE

Represents a Role object. This is not returned to the user at any time!

Roles are just permission bits (we currently use 32 bit ints for the nerds out there wondering). Each bit is either toggled on or off.

Role Name Notes
API_ACCESS The role to access the API endpoints (do not bother requesting for these)
ARRAY_ACCESS To access any endpoints that return an array
MANAGE_FILE To access POST, PATCH & DELETE file endpoints
MANAGE_URL To access POST, PATCH & DELETE url endpoints

URL

Represents a Url object

Field Data Type Notes
id Long The ID of the url
url String A non-null String containing the URL's domain (with any subdomains).
reason String A non-null String with the URL's blacklisting reason.
addition_date Long A non-null Long with the Epoch Millisecond of when this entry was added.
