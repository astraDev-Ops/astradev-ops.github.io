Errors


Possible errors and what they mean.

Error Message Explanation
FORM DATA INVALID A required field is missing or does not conform to the required data type
INVALID FORM RECEIVED A required field is missing or does not conform to the required data type
URL PROVIDED IS INVALID You sent a malformed URL, structure should be in the form of SUBDOMAINS.DOMAIN.TLD
SHA256 HASH PROVIDED IS INVALID You sent a malformed hash.
AT LEAST ONE VALID QUERY PARAMETER IS REQUIRED You sent a blank GET request with no query parameters.
MISSING REQUEST PARAMETER You sent a blank GET request with no query parameters.
INVALID REQUEST PARAMETER. You sent a GET request with the incorrect data type.
MISSING FORM ATTACHED TO REQUEST. You sent a POST/PATCH/DELETE with no JSON request body attached to it.
INVALID API KEY PROVIDED. The API key provided is incorrect or invalid.
MISSING API KEY. An API key header was not provided in the request.
MISSING AUDIT LOG HEADER. An audit log header was not provided in the request.
INVALID AUDIT LOG HEADER RECEIVED. An audit log header was supplied but did not meet the minimum character requirements.
YOU DO NOT HAVE PERMISSION TO ACCESS THIS ENDPOINT. The API key provided does not have authorisation to access the endpoint.
A DATABASE ERROR HAS OCCURRED. The server has failed to communicate with the database. THIS IS A SERVER ERROR.
A VALIDATION ERROR HAS OCCURRED. The server has failed to validate the request data. THIS IS A SERVER ERROR.


File


The File endpoint data with version specific information.

Field Data Type Minimum API Version Notes
id Long v1 The id within the database
sha256 String v3 A non-null 64 bit hex String with the file's sha256 hash
sha3_384 String v3 A nullable String with the file's sha3_384 hash.
classification String v3 A nullable String with the malware class (NULL IF POTENTIALLY NOT MALICIOUS).
mime_type String v3 A nullable String with the file MIME type (if known).
addition_date Long v3 A non-null long with the Epoch Millisecond of when this entry was added to the database.
detection_date Long v3 A non-null long with the Epoch Millisecond of when this entry was either first detected in the wild (if known) or the addition date.
sample_available Boolean v3 A non-null Boolean if there is a copy of the file locally.

Url


The Url endpoint data with version specific information.

Field Data Type Minimum API Version Notes
id Long v1 The id within the database
url String v1 A non-null String containing the URL's domain (with any subdomains).
reason String v1 A non-null String with the URL's blacklisting reason.
addition_date Long v2 A non-null long with the Epoch Millisecond of when this entry was added to the database.
