API


The api endpoint is for modifying authentication and authorisation of a user.

GET /api?query={ user.id }
Accounts accessing this endpoint require the following permissions: API_ACCESS

Get the API status of a user account.

Returns Status: 200 with one of the following JSON bodies:

"{message : NO ACCOUNT}"

"{message : ACTIVE}"

"{message : DISABLED}"

"{message : BANNED}"


POST /api
Accounts accessing this endpoint require the following permissions: API_ACCESS

Create a new API user account.

JSON Parameters
Field Data Type Notes
discord_id Long A non-null Long of the discord ID

Returns Status: 200 on successful creation OR Status: 400 if account exists already.


PATCH /api
Accounts accessing this endpoint require the following permissions: API_ACCESS

Reset and immediately invalid the old API key for an account.

JSON Parameters
Field Data Type Notes
discord_id Long A non-null Long of the discord ID

Returns Status: 200 with the api key. Example:

"{message : API_KEY_FOR_YOU_ENJOY_YOUR_NEW_KEY}"


DELETE /api
Accounts accessing this endpoint require the following permissions: API_ACCESS
This endpoint supports the audit log header (x-audit-reason).

If the audit log header is missing then the default action is to disable the API user. If the header is present the default action is to ban the API user.

JSON Parameters
Field Data Type Notes
discord_id Long A non-null Long of the discord ID

Returns Status: 200 on success, or in the case of a failed ban a Status: 400 will be returned.
